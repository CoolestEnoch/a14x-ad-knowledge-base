# 5.1 世界模型 in 自动驾驶

# 介绍

## 1.1 世界模型简介 （World Model，WM）

世界模型作为人工智能领域实现通用人工智能的核心理论框架，近年来因其突破性潜力备受关注——尤其在模拟环境动态与支持复杂决策方面。其本质在于通过构建物理/数字世界的内部表征机制，实现环境状态理解与未来演变预测，从而为智能体的感知、推理和行动提供基础支撑架构。

世界模型的研究起源可追溯至心理学领域的"Mental Model"理论。作为该理论的重要奠基人，Craik K.提出：人类通过构建外部世界的简化抽象表征，能够模拟现实运行规律并预测未来事件。此外，控制理论中模型预测控制 (MPC) 的基本逻辑充分覆盖了世界模型的核心节点。这些先驱工作证明了应用数学模型来理解和预测未来的潜力。Ha D.等人创新性地将心理模型理论引入强化学习框架，实现了从高维感官输入中提取潜在状态表征z<sub>t</sub>，并基于动作a<sub>t</sub>和历史隐状态h<sub>t</sub>预测未来环境演变z<sub>t+1</sub>的技术路径。这种端到端框架模拟了人类通过经验构建认知模型的过程，并实现了从虚拟推演到现实迁移的闭环，被认为是基于机器学习的世界模型的先驱

Yann LeCun于 2022 年发布的技术报告对世界模型的发展产生了深远影响。他在该文献中指出，大多数人都知道如何在诸多从未遇到过的情况下采取行动，但机器学习系统即便经过大量训练，在驾驶等现实世界任务中，仍无法达到人类的可靠性水平。而解决这一困境的答案 “或许在于人类及许多动物具备学习世界模型的能力 —— 即构建关于世界运行方式的内在模型的能力”。

Yann LeCun将自主智能（Autonomous Intelligence）与动物大脑相关联，认为世界模型是其中最复杂的部分，其他模块还包括：配置器模块（configurator module）、感知模块（perception module）、成本模块（cost module）、短期记忆模块（short-term memory module）以及行动者模块（actor module）。

随后，LeCun给出了联合嵌入预测架构（JEPA）和分层JEPA（H-JEPA），以指导世界模型的构建。JEPA在多个抽象级别和多个时间尺度下以分层方式学习表示感知和行动计划。这是一种符合人类直觉的设计，因为人类和许多动物能够设想多级抽象，通过将复杂的动作分解为较低级别的序列来执行长期预测和长期规划。LeCun的观念直接挑战了“纯数据驱动”的范式，强调机器需像人类一样，通过少量交互即可快速学习新技能，并且给出非单一的预测结果。

当前世界模型的前沿研究显著体现在Multi-modal Large Models (MLMs) 以及Embodied Artificial Intelligence (Embodied AI)。例如，Sora展示了生成符合物理世界基本原则的复杂视觉叙事的深刻能力。作为“世界模拟器”，Sora具有对动态世界的因果推理能力。Sora能生成长达60秒的动态视频，模拟3D一致性场景（如摄像机旋转中的物体运动），甚至预测物理交互（如食物咬痕）。潜在空间世界模型已广泛应用于Embodied AI agent。该范式将输入信息映射到潜在空间，并在潜在空间内操作以提取和利用高级语义信息，从而使机器人能够感知世界环境的基本表示(例如，I-JEPA，A-JEPA等)，更准确地执行具体化的下游任务(例如，STP，MuDreamer等)。

## 1.2 驾驶世界模型研究背景（Driving World Model，DWM）

当前，世界模型已广泛应用于自动驾驶领域，成为连接感知、决策与控制的核心工具。<font style="color:rgba(0, 0, 0, 0.85);">驾驶世界模型旨在为自动驾驶系统构建可解释、可预测的 “世界认知结构” 的模型。它能够让自动驾驶车辆不仅识别出周围环境中的物体，还能理解这些物体的含义以及预测接下来可能发生的事情，具备 “脑内模拟” 的能力，类似于人类通过经验积累形成对环境的理解模型，从而提前预判、灵活应变。</font>

<font style="color:rgba(0, 0, 0, 0.85);">自2010年至今，自动驾驶技术完成了从\`感知-决策-控制'独立框架到端到端一体化模型，再到世界模型理论的变迁。然而，端到端模型对于泛化能力和未来决策的需求仍然无法忽视。DWM与端到端模型的合作系统可以被视为一种分层决策系统：DWM提供决策的上下文边界，端到端模型在边界内实现最优执行，二者共同构成自动驾驶系统的智能化核心。从设计理念的维度看，端到端方案旨在实现传感器数据到控制指令的直接映射，而DWM侧重于学习数据空间的规律。在同时具有上述两者性质的系统中，两者一般表现为因果关系，即DWM为端到端模型提供历史特征表示与未来特征预测。</font>

## 1.3 研究意义

**场景理解和预测能力**：一些驾驶世界模型专注于生成逼真的未来场景和高精度的未来深度感知，有助于车辆预测其他道路使用者的行为、进行风险感知规划，从而提高自动驾驶的安全性与可靠性。

**提升自动驾驶系统的泛化能力**：驾驶世界模型试图为系统建立一种可迁移、可解释、可预测的 “世界理解力”，能够更好地适应不同的交通环境和场景变化，实现大规模泛化能力。

**更接近人类驾驶专家的决策**：一些世界模型的决策过程更接近人类司机的 “反复评估 — 微调 — 决策” 行为。

**加速仿真和训练过程**：世界模型可以作为虚拟环境，安全地简化和扩展自动驾驶机器的训练。它能够生成新的场景，包括行人、交通和道路状况等，帮助解决训练中的数据差距问题，减少对真实世界数据的依赖，降低训练成本，同时也能更高效地进行测试和优化。

## 1.4 主要挑战

\*\*统一任务：\*\*当前DWM在任务定义、基准测试与框架设计上呈现显著的非统一性。导致这种现状的原因是：传统模块化系统将感知、预测与规划拆解为独立子任务。端到端方法在统一架构任务上做出了巨大贡献。DWM的研究者希望其工作兼容上述所有任务，但是他们对"DWM应完成何种任务"具有认知差异。一部分研究聚焦像素级未来预测，另一部分则强调交通参与者轨迹建模。两者都缺乏对场景动态演变的全局理解。这种非统一性是技术演进的必经阶段，但同时阻碍了知识积累与学术交流，导致研究效率低下和资源浪费，以及降低了研究可信度。

\*\*可靠模拟器：\*\*DWM能够完成自动驾驶的多项子任务，但是仍然无法被称为‘可靠模拟器’，不仅仅是因为性能，延迟等客观原因。更重要的是当前的DWM难以满足驾驶模拟器所需的可信评估与可控干预。首先，很多DWM的工作主要关注开环场景下的质量评估\cite{}，而缺乏对模型在闭环场景中产生实际影响的评估。Open-loop planning可以在大规模离线数据集上快速运行，进行海量测试和消融实验，以快速迭代模型的设计。Open-loop planning还能避免在closed-loop planning中因控制模块缺陷（如过于激进或保守）而掩盖或夸大DWM本身性能的问题。然而open-loop planning无法评估模型预测对未来自身动作（或控制输出）所引发的环境反应的准确性，即反应性预测。也无法直接体现预测信息如何最终转化为安全、舒适的驾驶行为。而closed-loop planCning被认为是更接近真实世界场景的测试方式和对鲁棒性的最终检验。Closed-loop planning能够评估自动驾驶系统在动态交互环境中的决策能力。忽略任何一种测试，都将导致对模型能力认知的不完整，并可能掩盖潜在的风险或局限性。其次，许多方法缺乏对特定语义元素进行精细控制和局部修改的能力。这种能力体现了模型在复杂驾驶情景下进行干预式测试的应用价值。

\*\*效率：\*\*为了获得更鲁棒的输出，DWM的复杂度不断增加，这进一步提升了计算资源的需求。尽管DWM在许多自动驾驶任务中取得了进展，但在面对现实世界中的大规模部署任务时，平衡模型复杂性与实时性仍旧是巨大的挑战。

\*\*可解释性：\*\*在安全关键的应用中，模型的可解释性至关重要，因为可解释性是具身智能agent安全融入物理世界的前提之一。然而，DWM的“黑盒”特性使其难以提供对场景与决策的因果解释，即“为何生成了特定未来场景轨迹”或“为何做出了某项驾驶决策”。这种不透明性严重阻碍了研究者、监管者和车辆用户对DWM的理解与信任，并成为真实环境应用中的潜在风险源。几乎所有方法依赖注意力权重等间接、后验的手段，缺乏对内部因果机制的分析。研究者给出的因果解释通常仅关注显著特征，模块或最终输出结果，难以构建覆盖上下文且符合人类偏好的因果描述。例如，多数融合感知模块仅能输出显著的标注，却无法说明这些标注如何影响最终轨迹生成。
