# 3.16 毫米波 for 3D检测 代鑫光

# 车载毫米波雷达基础定义
****

**3D毫米波雷达的定义：**3D 毫米波雷达是采用调频连续波（FMCW）体制、基于多天线阵列实现测角的车载毫米波探测设备，工作频段以 24GHz、77GHz 为主。它可同时解算出目标** 径向距离（Range）、水平方位角（Azimuth）、径向速度（Doppler）**三类信息，因此称作 3D 毫米波雷达。天线基本是水平排列，垂直方向孔径很小、波束很宽，无法分辨高度 / 俯仰角，所有目标被 “压扁” 在水平面。

> **<font style="color:rgb(0, 0, 0);background-color:rgba(0, 0, 0, 0);">注</font>****：**3D 毫米波雷达的 “3D” 不等于 XYZ 三维空间坐标，而是指水平方位角、径向距离、径向速度；它**<font style="color:rgb(0, 0, 0);background-color:rgba(0, 0, 0, 0);">不具备垂直高度（俯仰角）探测能力</font>**，所有目标被投影在同一水平面；极坐标数据可转换为笛卡尔坐标，但**<font style="color:rgb(0, 0, 0);background-color:rgba(0, 0, 0, 0);">只能得到 x/y，缺失真实 z，并非完整三维，存在信息缺失</font>**。
>

**4D毫米波雷达：**在3D毫米波雷达的基础上增加了俯仰角 Pitch/测高能力

> **注：**4D 毫米波雷达转成 XYZ + 径向速度，已经消除了 3D 雷达那种 “缺 Z 压扁” 的信息缺失；几何维度上是完整的（3D 位置 + 1D 速度），但仍受方位角俯仰角分辨率、噪声、速度维度不完备的物理限制，在XYZ的表达上不等于真值。
>

径向速度在笛卡尔坐标系的表达：目标真实三维速度 (v_x, v_y, v_z) 不能单靠一帧雷达解出，需要多帧 / 融合 / 模型假设。



# 车载毫米波雷达在自动驾驶中的地位和融合方案






**主流车载毫米波雷达方案：**1前+4角；1前+2角；前雷达一般为4T4R/8T8R的4D毫米波雷达，角雷达一般为3D毫米波雷达。



**车载毫米波雷达发展趋势：**

分布式边缘→卫星式中央集中架构（2025–2026 爆发趋势）

**传统 Tier1 分布式边缘架构：**毫米波雷达会在硬件本地完成数据运算，生成稀疏目标列表以及轻量化点云数据，再将整理完毕的结果打包传输给整车主机。整套雷达算法属于零部件厂商的技术黑盒，车企主机厂无法介入底层的信号处理环节，难以自主优化相关算法逻辑。

**新一代端到端模式：**雷达原始数据全量输出，Tier1 只做硬件探头，输出**ADC 原始张量 /  RD 二维谱/ 原始点云**。底层信号处理 + 融合 + 感知神经网络全部交由主机在域控自研，形成「射频原始数据→域控端到端 BEV 模型→障碍物输出」全链路自研闭环。
