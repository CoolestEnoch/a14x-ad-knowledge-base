<font style="color:rgb(0, 0, 0);">数据形式</font>

<font style="color:rgb(0, 0, 0);">视觉和热像仪。视觉和热像仪拍摄的图像可以提供车辆周围环境的详细纹理信息。视觉相机对照明和天气条件很敏感，而热像仪对白天/夜间的变化更有力，因为它们检测的是与物体的热量有关的红外辐射。然而，这两种类型的相机都不能直接提供深度信息。平均精度</font>

<font style="color:rgb(0, 0, 0);">LiDARs。LiDARs（光探测和测距）以三维点的形式提供周围环境的精确深度信息。它们测量以一定频率发射的激光束的反射。LiDAR对不同的照明条件有很强的适应性，与视觉相机相比，受各种天气条件的影响较小，如雾和雨。然而，典型的激光雷达在物体分类方面不如相机，因为它们不能捕捉到物体的精细纹理，而且它们的点会随着遥远的物体而变得稀疏。最近，飞溅激光雷达被开发出来，可以产生类似于相机图像的详细物体信息。调频连续波（FMCW）LiDAR可以提供速度信息。</font>

<font style="color:rgb(0, 0, 0);">雷达。雷达（无线电探测和测距）发射无线电波被障碍物反射，测量信号运行时间，并通过多普勒效应估计物体的径向速度。它们对各种照明和天气条件具有很强的适应性，但由于其分辨率低，通过雷达对物体进行分类是非常具有挑战性的。雷达经常被应用于自适应巡航控制（ACC）和交通堵塞辅助系统[13]。</font>

<font style="color:rgb(0, 0, 0);">超声波：超声波传感器发出高频声波来测量到物体的距离。它们通常用于近距离物体检测和低速场景，例如自动停车[13]。由于传感特性，超声波在很大程度上受到空气湿度、温度或污垢的影响。</font>

<font style="color:rgb(0, 0, 0);">GNSS和高清地图。GNSS（全球导航卫星系统）通过全球卫星系统和接收器提供精确的三维物体位置。GNSS的例子有GPS、Galileo和GLONASS。GNSS首先被引入汽车，作为驾驶辅助功能的导航工具[13]，目前GNSS也与高清地图一起用于路径规划和自主车辆的自我车辆定位。</font>

<font style="color:rgb(0, 0, 0);">IMU和里程表。与上面讨论的捕捉外部环境信息的传感器（即 "外部感知传感器"）不同，惯性测量单元（IMU）和里程表提供车辆的内部信息（即 "本体感知传感器"）[13]。惯性测量单元（IMU）测量车辆的加速度和旋转率，而里程表则测量里程数。自20世纪80年代以来，它们一直被用于车辆动态驾驶控制系统中。它们与外感觉传感器一起，目前被用于自动驾驶的精确定位。</font>

<font style="color:rgb(0, 0, 0);">多模态数据集</font>

<font style="color:rgb(0, 0, 0);">尽管KITTI数据集[75]被广泛用于自动驾驶研究，但其记录条件的多样性相对较低：它是在卡尔斯鲁厄--德国的一个中型城市，只在白天和晴天记录。其他经过审查的数据集，如[60]、[78]、[79]、[82]、[87]-[89]都是在一个以上的地点记录的。为了增加照明条件的多样性，[60]、[80]-[82]、[82]、[84]、[86]、[88]-[92]收集了白天和夜间的数据，[93]考虑了全天的各种照明条件，包括日出、上午、下午、日落、夜晚和黎明。牛津数据集[74]和牛津雷达RobotCar数据集[85]是通过在牛津地区全年驾驶汽车来收集的。它包含不同天气条件下的数据，如大雨、夜晚、阳光直射和雪。其他包含不同天气条件的数据集有[60]、[86]、[88]、[89]。在[95]中，LiDAR被用作生成地面实况的参考传感器，因此我们不认为它是一个多模式的数据集。然而，记录条件的多样性是很大的，从黎明到夜晚，以及反射、雨和镜头ﬂare。跨季节数据集[96]强调了全年变化的重要性。然而，它只提供相机图像和用于语义分割的标签。同样，视觉定位挑战和相应的基准[97]涵盖了天气和季节的多样性（但没有引入新的多模式数据集）。最近的Eurocity数据集[88]是我们审查过的最多样化的数据集。它是在几个欧洲国家的不同城市记录的。所有的季节都被考虑在内，还有天气和白天的多样性。到目前为止，该数据集只有相机，其他模式（如LiDARs）已经公布。</font>

<font style="color:rgb(0, 0, 0);">数据集大小</font>

<font style="color:rgb(0, 0, 0);">数据集的规模从只有1,569帧到超过1100万帧不等。我们审查过的最大的带有地面真实标签的数据集是nuScenes数据集[89]，有将近1,400万帧。与计算机视觉界的图像数据集相比，多模态数据集仍然比较小。然而，在2014年和2019年之间，数据集的规模已经增长了两个数量级。</font>

<font style="color:rgb(0, 0, 0);">标签</font>

<font style="color:rgb(0, 0, 0);">大多数经过审查的数据集为2D对象检测和语义分割任务提供了地面真实值标签[60]、[75]、[88]、[90]–[93]。KITTI[75]还为各种计算机视觉问题标记了跟踪、光流、视觉测程和深度。BLV3D[80]为跟踪、交互和意图提供了标签。三维场景理解的标签由[60]、[75]、[79]–[84]、[89]提供。</font>

<font style="color:rgb(0, 0, 0);">根据数据集的重点，物体被标记为不同的类别。例如，[90]只包含人的标签，包括可区分的个体（标记为 "人"）、不可区分的个体（标记为 "人"）和骑自行车的人；[60]将物体分为五组，并提供25个精细的标签，如卡车、三轮车、交通锥和垃圾桶。Eurocity数据集[88]主要关注脆弱的道路使用者（主要是行人）。[77]提供了一个用于场所分类的数据集，而不是给物体贴标签。场景被分类为森林、海岸、住宅区、城区和室内/室外停车场。[78]提供了用于驾驶行为预测的车辆速度和车轮角度。BLV3D数据集[80]为互动和意图提供了独特的标签。</font>

<font style="color:rgb(0, 0, 0);">方法学；方法论</font>

<font style="color:rgb(0, 0, 0);">在设计用于多模态感知的深度神经网络时，需要解决三个问题：融合什么：应该融合哪些感知模式，以及如何以适当的方式表示和处理它们；如何融合：应使用哪些融合操作；何时融合：在神经网络的特征表示的哪个阶段，传感模式应该组合。在本节中，我们总结了基于这三个方面的现有方法。</font>

<font style="color:rgb(0, 0, 0);">融合什么</font>

<font style="color:rgb(0, 0, 0);">LiDAR和相机（视觉相机、热像仪）是文献中最常见的多模态感知的传感器。</font>

<font style="color:rgb(0, 0, 0);">2 相机图像：文献中的大多数方法采用来自视觉相机的RGB图像或来自热感相机的一种类型的红外图像（近红外、中红外、远红外）。此外，一些工作提取额外的传感信息，例如光流[120]、深度[61]、[125]、[126]或其他多光谱图像[91]、[125]。</font>

<font style="color:rgb(0, 0, 0);">六 挑战和未决问题</font>

<font style="color:rgb(0, 0, 0);">正如导论中所讨论的（参见第一节），开发深度多模态感知系统对于自动驾驶来说尤其具有挑战性，因为它对准确性、鲁棒性和实时性都有很高要求。来自物体检测或语义分割的预测通常被转移到其他模块，如机动预测和决策。一个可靠的感知系统是无人驾驶汽车在不受控制的复杂驾驶环境中安全运行的前提条件。在第三节和第五节中，我们已经总结了多模式数据集和融合方法。相应地，在本节中，我们讨论了多模态数据准备和网络架构设计的剩余挑战和开放问题。我们重点讨论如何在保证实时性的同时提高多模态感知系统的准确性和鲁棒性。我们还讨论了一些开放性问题，如评价指标和网络架构设计。表一总结了挑战和开放性问题。I总结了这些挑战和开放性问题。</font>

<font style="color:rgb(0, 0, 0);">A. 多模态数据准备</font>

<font style="color:rgb(0, 0, 0);">1）数据的多样性。在一个复杂的任务上训练一个深度神经网络需要大量的数据。因此，使用具有不同驾驶条件、物体标签和传感器的大型多模式数据集可以极大地提高网络的准确性和对变化环境的鲁棒性。然而，由于成本和时间的限制以及硬件的限制，获取真实世界的数据并不是一件容易的事。开放的多模态数据集的大小通常比图像数据集的大小小得多。作为比较，KITTI[6]只记录了80,256个对象，而ImageNet[163]提供了1,034,908个样本。此外，这些数据集通常是在有限的驾驶场景、天气条件和传感器设置中记录的（更多细节在第三部分提供）。对象的分布也非常不平衡，被标记为汽车的对象比人或自行车的对象多得多。因此，用这些公共数据集训练的深度多模态感知系统在部署到非结构化环境中时的表现是值得怀疑的。</font>

<font style="color:rgb(0, 0, 0);">克服这些限制的方法之一是通过模拟进行数据扩充。事实上，最近的一项工作[164]指出，KITTI数据集中物体检测的大部分性能提升是由于数据的增强，而不是网络结构的进步。</font>

<font style="color:rgb(0, 0, 0);">2）数据质量和对齐：除了数据多样性和训练数据集的大小外，数据质量也会显着影响深度多模态感知系统的性能。 训练数据通常由人工标注者标记，以确保高标注质量。 然而，人类也容易犯错误。 图 11 显示了训练对象检测网络时标记过程中的两个不同错误。 与使用确定性预标签的有偏标签相比，当标签错误随机分布时，网络对标签错误的抵抗力要强得多。 [188]-[191] 进一步研究了带有错误标签的训练网络。 在 [192]、[193] 中研究了弱标签或错误标签对基于深度学习的语义分割性能的影响。 在[194]、[195]中讨论了标记错误对目标检测准确性的影响。</font>
