### 训练

* 单卡训练

`python tools/train.py configs/pointpillars/hv_pointpillars_secfpn_6x8_160e_kitti-3d-3class.py --gpu-id 0 --work-dir ./works_dir/train >> ./out_dir/2022.5.21_2_pointpillars.out&`

config文件是必选参数

gpuid默认为0 需要手动指定为空闲gpu

work-dir为工作目录，会生成tensorboard的相关文件以及权重文件和日志等信息

out\_dir 指定日志生成目录

* 多卡训练

<code>CUDA_VISIBLE_DEVICES=6,8 nohup tools/dist_train.sh configs/pointpillars/hv_pointpillars_secfpn_6x8_160e_kitti-3d-3class.py 2 --resume-from ./works_dir/train/latest.pth --work-dir ./works_dir/train >> ./out_dir/pointpillars.out& </code>

CUDA\_VISIBLE\_DEVICES是指定的gpu，指定几个就需要在configs参数后指定相应的线程，例中指定两张卡就需要2线程。

nohup 命令为不挂断地运行命令，可以理解为后台运行不需要一直开着teminal，期间可以在日志文件夹查看训练情况(out\_dir目录)

\--resume-from 是用来断点训练的，从0训练不需要填入（单卡也可以断点训练）

### 训练

<font style="color:#494949;">使用OpenPCDet训练网络，是运行tools目录下运行train.py，具体例如，运行</font>

<code><font style="color:#494949;">python train.py --cfg_file cfgs/kitti_models/pointpillar.yaml</font></code>

<font style="color:#494949;">其中</font><code><font style="color:#494949;">--cfg_file</font></code><font style="color:#494949;">参数是需要训练的网络的配置文件。</font>

<font style="color:#494949;">或者</font>

<code><font style="color:#494949;">CUDA_VISIBLE_DEVICES=0,1,2,3 python -m torch.distributed.launch --nproc_per_node=4 train.py --launcher pytorch --batch_size 8 --extra_tag baseline --cfg_file cfgs/kitti_models/pointpillar.yaml</font></code>

<font style="color:#494949;"></font>

<font style="color:#494949;">其中</font><code><font style="color:#494949;"> CUDA_VISIBLE_DEVICES=0,1,2,3 </font></code><font style="color:#494949;">表明使用gpu 0,1,2,3</font>

<font style="color:#494949;"></font>

<code><font style="color:#494949;">python -m torch.distributed.launch</font></code><font style="color:#494949;"> 使用torch.distributed.launch工具进行分布式训练</font>

<font style="color:#494949;"></font>

<code><font style="color:#494949;">--nproc_per_node=4</font></code><font style="color:#494949;">表明使用4个gpu</font>

<font style="color:#494949;"></font>

<code><font style="color:#494949;">train.py</font></code><font style="color:#494949;"> 运行的是train.py 这个文件</font>

<font style="color:#494949;"></font>

<code><font style="color:#494949;">--launcher pytorch</font></code><font style="color:#494949;"> 使用的是pytorch架构</font>

<font style="color:#494949;"></font>

<code><font style="color:#494949;">--batch_size 8</font></code><font style="color:#494949;"> 表明 bs=8</font>

<font style="color:#494949;"></font>

<code><font style="color:#494949;">--extra_tag baseline</font></code><font style="color:#494949;"> 给此次训练加一个额外标签叫做baseline，这个会在输出的工作路径下创建一个baseline文件夹，所有生成的文件均在里面</font>

<font style="color:#494949;">此时任务运行时输出会自动重定向到nohup.out，使用 tail -f nohup.out 即可看到实时输出。也可使用 > 重定向到任一文件。</font>

# 框架训练常见的一些问题

## 报错 ：no module name 'pcdet'

在容器内编译后，运行代码出现上面的提示，因此需要在容器内说明一下，执行以下命令：

`export PYTHONPATH=$HOME/Cicv_task1_223/:$PYTHONPATH`

即可正常运行

## libGL.so.1

提示:libGL.so.1: cannot open shared object file: No such file or directory

在镜像环境内

apt update

apt install libgl1-mesa-glx

## KeyError: 'lidar\_idx'

把数据放到根目录下的data里

例如：数据集原来不在OpenPCDet/data中，现在把数据集OpenPCDet/data中

原因：待解决

## KeyError: 'road\_plane'

plane数据没有拷贝

解决方法1：拷贝进来plane数据

解决方法2：在算法的ymal文件中把`USE_ROAD_PLANE`改为`False`

## module 'spconv' has no attribute 'SparseModule'

是spconv版本迭代导致的错误

找到报错位置的文件将头文件

`import spconv`

改成

`import spconv.pytorch as spconv`
